Quarto Overview

  • Open-source scientific and technical publishing system
  • Created by the same group that created Rmarkdown
  • Combines the functionality of R Markdown, bookdown, distill, xaringian, etc into a single consistent system with “batteries included”
  • Publish reproducible, production quality articles, presentations, dashboards, websites, blogs, and books in HTML, PDF, MS Word, ePub, and more.
  • Multi-language support
  • Can support SAS via Jupyter

How it works

If rendering via RStudio, then knitr is used instead of jupyter.

Setup Guide

Software Installation

  1. Quarto
  2. If rendering SAS output you will need to install Python
    1. You can get it here. If you go this route, you will have to install Jupyter later (See point 3.2 of the Python package installation instrutions below for details.).
    2. Alternatively you can install Anaconda which includes Python and Jupyter.
  3. A tool to create and render Quarto docs
    1. R Studio
    2. VS Code

R Packages

If you plant to use RStudio to create and render Quarto files, you’ll need to install the rmarkdown (this will also install knitr) and quarto packages.

VS Codes Extenstions

If you plan to use VS Code to create and render Quarto file, you’ll need to install the Quarto and Python extensions from within VS Code. They also have extensions for SAS, R, and Jupyter but those are not needed for Quarto

Update Environmental Variables

  1. Press Win + S to open the search bar.
  2. Type Environment Variables and select Edit the system environment variables from the results.
  3. In the System Properties window, click the Environment Variables button near the bottom.
  4. Add new PATH variables
    1. In the Environment Variables window, under User variables, find the Path variable and select it. Click Edit to open the Edit Environment Variable dialog.
    2. In the Edit Environment Variable window, click New and type or paste the following 2 paths. Each entry in the PATH variable should be on a new line.
      1. Location where your python.exe is located. Mine is located in C:\Users\username\AppData\Local\anaconda3.
      2. Location where your pip (Python package installer) installation is located. Anaconda installs pip by default; mine is located at C:\Users\username\AppData\Local\anaconda3\Scripts. If you installed Python from a source other than Anaconda you will likely need to install pip after you have included python.exe in your PATH and then add the proper path as shown here. See point 3.1 of the Python package installation instrutions below for details on how to install.
    3. Click OK to close each window.
  5. Define new Environmental Variables
    1. In the Environment Variables window, under User variables, click New.
    2. Add the following two variables:
      1. QUARTO_PYTHON path.exe
      2. QUARTO_R C:\Program Files\R\R-4.4.1\bin\x64
        1. If you installation is in the C:\Users\username\AppData\Local folder, you can skip this step.

Python Package Installation

  1. Press Win + S to open the search bar.
  2. Type Command Prompt and select from the results.
  3. Type in the following commands, as needed:
    1. Ensure pip is installed: py -m ensurepip –upgrade
    2. If you did not install Anaconda, install Jupyter notebook, a web-based python notebook: pip install notebook
    3. Install SASPy, a package that provides interfaces between Python and SAS: pip install saspy
    4. Install SAS kernel, Python language kernel: pip install sas_kernel
    5. Install LaTeX to render PDF files: quarto install tinytex --update-path
      1. –update-path should add this to your PATH variables. If not, you’ll need to add manually.

Update the SASPy configuration file

  1. To locate your sascfg.py configuration file, type the following lines into a Command Prompt
    python
    import saspy
    saspy.SAScfg
    exit()
  1. Navigate to the path give. Mine is under C:\Users\username\AppData\Local\anaconda3\Lib\site-packages\saspy\sascfg.py.
  2. Create a copy of the file and name it sascfg_personal.py
  3. Open sascfg_personal.py and make the following change:
    1. Find the following line in your sascfg.py file: SAS_config_names=['default'] and modify to match the following: SAS_config_names=['winlocal'].
  4. You can also change the ODS sytle to your preferred style but this will only impact notebooks rendered via Jupyter. The default is HTMLBlue.

Check Installation

  1. Check Quarto setup
    1. Press Win + S to open the search bar.
    2. Type Command Prompt and select from the results.
    3. Type in the following command: quarto check
    4. Ensure all componentes are identified and no errors shown.
